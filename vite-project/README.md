# Blue Team Trainer — Vite Build

This is the alternative path for running the frontend if your analyst laptops cannot run Python (used by the simpler `start-trainer.ps1` / `start-trainer.sh` launchers in the parent folder).

The Vite production build is fully static — the resulting `dist/` folder works offline with no runtime dependencies. You can open `dist/index.html` directly in a browser, or serve it with any static file server.

## Workflow

### Once, on a build machine with internet

```bash
cd ./BlueTeam-Trainer/vite-project
npm install
npm run build
```

This produces `vite-project/dist/` containing `index.html`, plus an `assets/` folder of bundled JS/CSS. Total size ~150 KB gzipped.

### Distribute to air-gapped analyst laptops

Copy the `dist/` folder to each laptop. Either:

- **Double-click `dist/index.html`** — works in Firefox out of the box. Chrome and Edge may block module loads from `file://`; if so use one of the next two options.
- **Drop into any web server** the analyst already has (IIS, nginx, Apache, Caddy).
- **One-liner serve** (any of these work — pick whichever language the laptop has):
  ```bash
  cd ./BlueTeam-Trainer/vite-project/dist
  python -m http.server 8080          # Python
  npx serve -p 8080                    # Node.js
  php -S 127.0.0.1:8080                # PHP
  ```

Then open `http://localhost:8080`.

### Development with hot-reload

If you want to extend the trainer (add techniques, change the UI, etc.):

```bash
cd ./BlueTeam-Trainer/vite-project
npm install
npm run dev
```

This runs Vite's dev server on `http://127.0.0.1:5173` with hot module reload. Edit `src/BlueTeamTrainer.jsx` and the browser updates instantly. When you're done, run `npm run build` to produce a new shippable `dist/`.

## Project layout

```
vite-project/
├── package.json              # Dependencies + scripts
├── vite.config.js            # Vite config (relative base for portability)
├── index.html                # Entry HTML
├── src/
│   ├── main.jsx              # React mount point
│   └── BlueTeamTrainer.jsx # The component (copy of root JSX)
└── dist/                     # Generated by `npm run build`
```

## Why this exists

The parent folder's `blueteam-trainer.html` is the simpler path — just a single HTML file plus a few vendor JS files. It transforms JSX in the browser at load time, which means there's no build step at all but a slight (~1s) startup delay.

The Vite path is for cases where:
- You can't install Python on analyst laptops
- You don't want any runtime JSX transformation overhead
- You're customising the trainer significantly and want a proper development workflow
- Your security team prefers a static build artifact over runtime-compiled code

Both paths render the exact same component and produce the exact same training experience.
